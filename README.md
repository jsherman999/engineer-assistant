# Engineer Assistant

A native macOS (SwiftUI) app that teaches a high-school STEM student about MacOS and Linux. Two modes:

- **Ask Mode** — open-ended, streaming Q&A with Claude about sysadmin, coding, MacOS, or Linux.
- **Course Mode** — name a subject, get a short interactive course (generated by Claude via tool-use, cached on disk) with hands-on exercises in a **real sandboxed shell** and auto-graded challenges.

Every chat message and shell keystroke is recorded to an append-only event log for a parent or instructor to review.

### Notable implementation choices (differ from the original PLAN)

- **Event store is append-only JSONL** (`events.jsonl`), not SQLite/GRDB. All events are captured and fully reconstructible; this avoids a native dependency. May migrate to SQLite if the dashboard needs SQL queries.
- **Terminal is a real PTY** rendered by [SwiftTerm](https://github.com/migueldeicaza/SwiftTerm). Per-command exit codes/stdout are captured via shell-integration markers (zsh `preexec`/`precmd`, bash `PROMPT_COMMAND`) that are stripped before display.
- **Container runtime auto-detects and prefers Apple's `container`** (best on macOS 26+ Apple Silicon), then Podman, then Docker — all behind one abstraction.

## Requirements

- Apple Silicon Mac (M1 or newer)
- macOS 14+ (macOS 26+ recommended for Apple `container`)
- Swift toolchain (Xcode 15+ or Command Line Tools)
- An Anthropic API key (stored in the macOS Keychain)
- For **Linux courses**, a container engine — pick one:
  - **Apple `container`** (recommended on macOS 26+): `brew install container && container system start && container system kernel set --recommended`
  - or `brew install podman && podman machine init && podman machine start`
  - or Docker Desktop

macOS courses need no container engine.

## Building & running

```sh
swift build
swift run EngineerAssistant
```

Or open `Package.swift` in Xcode and run the `EngineerAssistant` scheme.

On first launch, open **Settings** (gear icon) and paste your Anthropic API key — it is stored only in the macOS Keychain. Then pick **Ask** or **Course** at the top of the chat. Generated courses appear under the **Courses** button and resume where you left off.

## Testing

```sh
swift test
```

The default suite is hermetic. Container integration tests are gated:

```sh
EA_RUN_CONTAINER_TESTS=1 swift test --filter ContainerIntegrationTests
```

## Data locations

```
~/Library/Application Support/EngineerAssistant/
├── events.jsonl        -- append-only chat + shell + lesson event log
├── progress.json       -- per-course resume point
├── results.json        -- per-course/lesson saved challenge results (attempts kept across retakes)
├── students.json       -- student<N> counter + the dirs each course has allocated
└── courses/            -- generated, cached course JSON
```

Sandbox working dirs live under the user's home as `~/students/student<N>` so the shell shows a
short, friendly path. A fresh `student<N>` is allocated every time a course is opened (the counter
only ever climbs); purging a course removes all the dirs it allocated.

Keychain holds `anthropic_api_key`.
